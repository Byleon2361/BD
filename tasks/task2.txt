1. Установите следующие параметры сервера: maintenance_work_mem=128MB
max_connections=200
2. Перезапустите экземпляр.
3. Установите стандартную схему pgbench, измерьте ее размер.
4. Запустите нагрузочный тест с помощью  pgbench.
5. Спровоцируйте блоатинг таблиц схемы pgbench, измерьте размер таблиц и индексов pgbench.
6. Повторите нагрузочный тест.
7. Посмотрите процессы экземпляра.
8. Выполните аварийное завершение экземпляра, запустите его вновь, найдите записи в журнале.
9. Запретите подключение приложения.
10. Выполните перестроение объектов pgbench, измерьте размер таблиц и индексов pgbench.
11. Разрешите подключение приложений.
12. Повторите нагрузочный тест.

Должны быть усатновлены
docker
docker-compose
postgresql-client

Нужно разaрхевировать бд

Поднятие бд
docker-compose up -d --build

Проверка памяти
❯ docker exec -it bd-postgres-1 psql -U postgres -d demo_big -c "SHOW maintenance_work_mem;"

войтв в бд
docker exec -it bd-postgres-1 psql -U postgres -d demo_big

ALTER SYSTEM SET maintenance_work_mem = '128MB';
ALTER SYSTEM SET max_connections = 200;
