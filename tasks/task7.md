Сделать 3-5 ручек для вызова через http клиент, которые вернут результат выполнения запросов.

Примеры запросов:
# 1. Главная страница
curl http://localhost:3000/

# 2. Топ новостей
curl "http://localhost:3000/api/news/top?limit=5"

# 3. Статистика по категориям
curl http://localhost:3000/api/stats/categories

# 4. Поиск
curl "http://localhost:3000/api/news/search?q=technology"

# 5. Топ авторов
curl "http://localhost:3000/api/authors/top?limit=5"

# 6. Ежедневная статистика (из витрины)
curl http://localhost:3000/api/stats/daily

# 7. Health check
curl http://localhost:3000/api/health


--------------------------------------------------------------------------------
Что именно означают эти ответы от curl

curl http://localhost:3000/
→ Главная страница API. Показывает приветствие и список всех доступных эндпоинтов.
Это доказывает, что сервер запущен и готов к работе.
curl http://localhost:3000/api/stats/categories
→ Статистика по категориям новостей за последние 30 дней:
Сколько статей в каждой категории
Общее количество просмотров и лайков
Среднее количество просмотров
Engagement rate (вовлечённость)
Данные реальные — из твоей базы (health лидирует по просмотрам).

curl http://localhost:3000/api/stats/daily
→ Витрина данных (data mart) — детальная ежедневная/посуточная статистика.
Возвращает 20 самых свежих или популярных новостей с полными полями:
Заголовок, содержание, категория, автор, метрики, теги, дата публикации
Это показывает, что data-mart.js успешно отработал и создал агрегированную коллекцию.

curl "http://localhost:3000/api/news/top?limit=5"
→ Топ-5 новостей по количеству просмотров.
Возвращает статьи с максимальными metrics.views.
Работает отлично — сортировка по просмотрам, лимит 5.
curl http://localhost:3000/api/health
→ Проверка здоровья системы:
Подключение к MongoDB: connected
База: news_aggregator
Коллекций: 6
Объектов (документов): 3187
Это доказывает, что всё живое и данные загружены.

curl "http://localhost:3000/api/news/search?q=technology" → пока ошибка
→ "text index required" — потому что текстовый индекс ещё не создан.
Это единственная мелочь, которую можно доработать (см. ниже).
